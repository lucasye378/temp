# FINANCE-REPORT-SERVICE-FEEDBACK-PATH-V1.md

## Goal

为“财经报道拆解服务”定义第一次真实反馈验证路径。

这一步的目标不是立即大规模外发，
而是先回答清楚：

- 第一批适合发给谁
- 发什么
- 希望回收什么反馈
- 什么结果算值得继续

---

## 1. First validation object

### 交付主题
**单条财经新闻 / 财经报道 → 一张结构化拆解卡**

### 交付文件
- `FINANCE-REPORT-SERVICE-SAMPLE-V1.md`

---

## 2. Target recipient type v1

第一轮适合发给这类对象：

- 对金融市场有兴趣，但经常看新闻半懂不懂的人
- 想先提升理解能力，而不是直接要投资建议的人
- 愿意给一句到几句真实反馈的人

一句话：

**不是发给所有看财经的人，而是发给“确实卡在看不懂市场语境”的人。**

---

## 3. Sending rule v1

### 首轮发送规模
先按 **1 个对象** 设计。

原因：
- 先验证用户是否真的感知到“这不是普通财经摘要”
- 先验证反馈问题是否清楚
- 避免在错误版本上批量发送

### 首轮发送顺序
1. 发送财经报道拆解卡正文
2. 附上 3–4 个最小反馈问题
3. 等回复后，再决定是否发给第二个对象

---

## 4. Feedback questions v1

如果要发，最小反馈问题只问：

1. 看完后，你是否更明白这条新闻到底在说什么了？
2. 这张卡有没有帮你更清楚地区分“事实”和“解释 / 市场反应”？
3. 这张卡相比普通财经摘要，有没有更强的“终于看懂了”感？
4. 这张卡相比你自己硬看财经新闻，省事多少？

---

## 5. Success threshold v1

如果首个对象反馈里命中以下 3 条及以上，
就值得继续发第二个对象：

- 更明白新闻到底在说什么
- 更清楚地区分事实与解释
- 明显感觉比普通财经摘要更有帮助
- 主观觉得明显省事
- 愿意继续看类似拆解卡

如果命中不足 3 条，
先不要扩大发送，
而是先改：
- 拆解结构
- “市场为什么在意”部分
- “现在还不能乱推什么”部分

---

## 6. Non-goals

这一轮先不做：
- 多对象并行发送
- 投资建议测试
- 定价测试
- 周期性服务版本

先把：

**1 个对象 → 1 次真实反馈 → 1 次判断**

跑通就够。

---

## One-Sentence Rule

**财经报道拆解服务的第一次真实验证，不是证明所有财经读者都需要它，而是先验证“看得懂标题但看不懂市场语境的人”是否觉得它比普通财经摘要更有帮助。**
