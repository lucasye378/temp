# READING-DIGEST-FEEDBACK-PATH-V1.md

## Goal

为“知识压缩 / 阅读精华提炼服务”定义第一次真实反馈验证路径。

这一步的目标不是立即大规模外发，
而是先回答清楚：

- 第一批适合发给谁
- 发什么
- 希望回收什么反馈
- 什么结果算值得继续

---

## 1. First validation object

### 交付主题
**高信息密度长内容 → 一页精华提炼卡**

### 交付文件
- `READING-DIGEST-SAMPLE-V1.md`

---

## 2. Target recipient type v1

第一轮适合发给这类对象：

- 经常想读长内容，但没时间完整读完的人
- 更关心“什么值得记住”，而不是“全文大概讲了什么”的人
- 愿意给一句到几句真实反馈的人

一句话：

**不是发给所有读者，而是发给“确实有内容压缩需求”的人。**

---

## 3. Sending rule v1

### 首轮发送规模
先按 **1 个对象** 设计。

原因：
- 先验证用户是否真的感知到“这不是普通摘要”
- 先验证反馈问题是否清楚
- 避免在错误版本上批量发送

### 首轮发送顺序
1. 发送精华提炼卡正文
2. 附上 3–4 个最小反馈问题
3. 等回复后，再决定是否发给第二个对象

---

## 4. Feedback questions v1

如果要发，最小反馈问题只问：

1. 看完后，你是否更快抓住了这篇内容里真正值得记住的部分？
2. 这张卡相比普通摘要，有没有更强的“值得保存 / 复看”感？
3. 你接下来更可能做什么？
   - 直接保存这张卡 / 想去读原文 / 没什么动作
4. 这张卡相比你自己硬读，省事多少？

---

## 5. Success threshold v1

如果首个对象反馈里命中以下 3 条及以上，
就值得继续发第二个对象：

- 更快抓住重点
- 明显感觉比普通摘要更值得保存
- 愿意保存 / 复看
- 主观觉得明显省事
- 愿意继续收类似卡片

如果命中不足 3 条，
先不要扩大发送，
而是先改：
- 卡片结构
- “为什么值得记住”部分
- 与普通摘要的差异表达

---

## 6. Non-goals

这一轮先不做：
- 多对象并行发送
- 定价测试
- 销售文案优化
- 自动化生产流程

先把：

**1 个对象 → 1 次真实反馈 → 1 次判断**

跑通就够。

---

## One-Sentence Rule

**阅读精华提炼服务的第一次真实验证，不是证明所有人都需要它，而是先验证“内容压缩需求真实存在的人”是否觉得它比普通摘要更值得保存。**
