# PROJECT-IDEAS.md

## Real Task Candidate Chosen for Closed-Loop Test

### Task

建立第一版“低编程门槛、有现实价值、可快速验证”的项目候选清单。

### Why this is a real task

- 它不是纯框架任务，而是直接面向 Goal 2（做出有价值的项目）
- 它会产出一个可供后续筛选和验证的真实候选列表
- 它会逼迫 Allen 使用当前 v1 系统来完成：记录、思考、queue 更新、follow-up

## First Candidate List v1

### 1. 书籍 / 长文精华提炼服务
- Value: 帮没有时间的人快速吸收高价值内容
- Startup cost: low
- Validation path: 先人工做几份样例，测试是否有人愿意付费

### 2. 商品筛选 / 性价比推荐服务
- Value: 帮人减少比价和筛选成本
- Startup cost: low
- Validation path: 针对具体品类先做 shortlist 服务

### 3. 第一轮研究 / 决策包服务
- Value: 帮人快速完成复杂主题的第一轮研究
- Startup cost: low
- Validation path: 先做定制型服务而非产品

### 4. 复杂信息整理与行动化服务
- Value: 把散乱资料、聊天、网页整理成行动清单
- Startup cost: low
- Validation path: 先给小团队或个人做“整理包”

### 5. 行业信息高价值摘要服务
- Value: 帮人过滤噪音、只保留重要更新
- Startup cost: low
- Validation path: 先聚焦一个具体领域

### 6. 政策 / 文件人话解读服务
- Value: 降低阅读门槛，提高理解速度
- Startup cost: low
- Validation path: 先做高频复杂文件场景

### 7. 服务 / 工具方案对比服务
- Value: 帮用户在多个工具或方案中做选择
- Startup cost: low
- Validation path: 聚焦单一选择场景

### 8. 创作者资料整理与选题支持
- Value: 降低创作者准备成本
- Startup cost: low
- Validation path: 先做一类创作者的支持包

## First Screening Lens

每个候选方向先用 4 个问题筛：
1. 低效是否真实且高频？
2. Allen 是否能在低编程条件下立即介入？
3. 用户是否能明确感知价值？
4. 是否能先人工 / 半自动验证付费意愿？

## Current Judgment

当前最值得优先进一步推进的 3 个候选是：
1. 书籍 / 长文精华提炼服务
2. 商品筛选 / 性价比推荐服务
3. 第一轮研究 / 决策包服务
