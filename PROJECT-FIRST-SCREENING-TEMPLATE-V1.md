# PROJECT-FIRST-SCREENING-TEMPLATE-V1.md

## Goal

定义一个项目从 `incubating` 进入更高优先级判断前的第一轮筛选模板。

它的作用不是替代完整决策，
而是先回答：

**这个项目值不值得继续往前推一步？**

---

## When to use

适用于这些场景：
- 有一个新项目想法刚进入 queue
- 某个 incubating 项看起来开始值得升级
- Allen 需要在多个候选方向中先做第一轮粗筛

不适用于：
- 已经进入真实交付阶段的项目
- 需要完整研究或用户访谈后才能判断的问题

---

## First screening questions v1

每个项目先回答下面 6 个问题：

### 1. 低效问题是否真实存在？
不是“听起来可能有需求”，
而是现实里真的有人反复卡在这里。

### 2. 用户是否能明确感知价值？
用户拿到结果后，
是否会明显感觉：
- 更省时间
- 更省脑力
- 更快完成第一次选择 / 第一次动作

### 3. Allen 能否在低编程条件下立即介入？
如果必须先搭重系统、写大量代码、接复杂基础设施，
就不适合作为当前优先项目。

### 4. 能否在短周期内做出最小样例？
如果连第一份样例都很难在短时间内做出来，
就不适合优先推进。

### 5. 是否容易获得真实反馈？
不是理论上“可能有人会喜欢”，
而是现实里是否存在：
- 能接触到的对象
- 能给反馈的人
- 能形成最小验证闭环的路径

### 6. 这个项目是否值得占用当前主链？
即使方向不错，
也要问：

**它现在是否比其他候选更值得占用唯一 active 位？**

---

## Simple scoring rule v1

每题先做三档判断：
- 强
- 中
- 弱

### 初步规则
- **4 个以上“强”**：值得继续推进到下一轮定义
- **2–3 个“强”**：先保留在 incubating / ready 观察
- **0–1 个“强”**：暂不推进

---

## Output format v1

第一轮筛选输出只需要包含：
- 项目名称
- 6 个问题的简答判断
- 一句话结论：推进 / 保留 / 暂不推进
- 如果推进，下一步最小动作是什么

---

## One-Sentence Rule

**第一轮项目筛选，不是证明项目一定会成，而是判断它是否值得再往前推一个最小运行单元。**
