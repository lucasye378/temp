# DECISION-PACK-SAMPLE.md

## Minimal Sample Task v1

### Task Name
首个“第一轮研究 / 决策包”样例任务

### Chosen Scenario
**帮助一个用户在 3 个可能方向中选择最值得先验证的项目方向。**

这个样例与 Allen 当前真实工作高度一致，因此既能作为服务样例，也能直接服务于内部项目探索。

### User Need
用户面对多个可能方向时，通常会遇到：
- 信息散
- 方向多
- 缺少第一轮比较
- 不知道先试哪个

### Sample Deliverable
一个最小决策包应包含：
1. 候选方向列表
2. 每个方向的核心价值
3. 启动成本 / 现实阻碍
4. 为什么优先选其中一个
5. 推荐的下一步动作

### Why this is a good first sample
- 与 Allen 当前能力高度匹配
- 不需要重开发
- 可以马上产出一版完整样例
- 直接检验“研究、比较、判断、建议”这套能力是否有价值

### Immediate Next Step
用现有 `PROJECT-IDEAS.md` 为基础，产出一版更像交付物的“最小决策包”，而不是只停留在候选清单。

### Success Criteria
- 产出一份结构清楚的最小决策包
- 能说明为什么选某个方向优先推进
- 包含具体的下一步动作建议
- 可以被视为未来付费服务的样例雏形
